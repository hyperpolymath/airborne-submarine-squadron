This repo now hosts a playable Gossamer frontend that wraps the AffineScript/WASM core plus the CLI/browser launcher ecosystem, with a focus on finishing the Orbit/depth-charge gameplay loop while stabilizing builds and installs.

== Status

This is a from-scratch rewrite. The project now ships a deterministic update loop, HUD telemetry, Gossamer loader/controls, weapons+depth charges, leaderboard tracking, and a launcher/installer pipeline. The current goal is finishing the AffineScript build path while improving deployment and multi-platform packaging.

== Quick Start (WASM)

Prereqs:

- AffineScript compiler available on PATH (or set `AFFINESCRIPT_REPO` to a local checkout)
- Node.js (for the simple WASM runner)

[NOTE]
====
`./build.sh` prefers a compiler but will copy `dist/airborne-submarine-squadron.wasm` into `build/` (and the legacy `airborne-final-working.wasm`) when no compiler is present, so you can still launch the web or Gossamer front end without a full rebuild.
====
[source,bash]
----
# Build to WebAssembly
./build.sh

# Run the WASM module (exports `main`)
node run_wasm.js build/airborne-submarine-squadron.wasm main
----

== Launcher Quick Start

[source,bash]
----
./launcher.sh --browser     # start the browser build (Deno file server + web HUD)
bash gossamer/launch.sh     # run the Gossamer desktop build (fallback uses Deno + local webview)
./launcher.sh --stop        # stop any running Deno/Gossamer instances
----

== Design Principles

- Text-first simulation: make the loop observable before adding graphics.
- Deterministic updates: same inputs produce the same outputs.
- Small, explicit data structures: no hidden mutation.
- Make safety constraints explicit in types and data flow.

== Build Notes

`build.sh` prefers a global `affinescript` binary. If not found, it will use a local compiler via `AFFINESCRIPT_REPO=/path/to/affinescript`.
If no compiler is available and the repository has a prebuilt WASM in `dist/airborne-submarine-squadron.wasm`, the script copies that into `build/` so the launcher can run without a rebuild.

== Project Highlights

- Gossamer loading/controls with retro battle background, boot-phase gating, and on-screen legend showing depth charges, afterburner, and key bindings.
- Orbit-ready backup mode, networked leaderboard, diver/eject mechanics, depth charges, chaffs/mines, and custom sub skins (rainbow/pride/spectrum).
- Launcher/installer combo that can start/stop Deno browsers, Gossamer fallback, tray integration, and desktop shortcuts, plus a build script that falls back to `dist/` if AffineScript is missing.
