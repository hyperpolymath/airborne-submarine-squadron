== Burble/Groove Event Boundary

*Authoritative event ownership:* game loop + Gossamer event bus.

Burble is used for:

* Voice-first communication.
* LLM support channel for operational/development coordination.
* RTSM high-bandwidth offload where needed.

Burble is *not* the authoritative gameplay event-state owner.
If event mirroring is needed for multiplayer/admin/ops, mirror selected events over
Groove/API/FFI with explicit timestamps and IDs, while canonical game state stays local
and deterministic.
